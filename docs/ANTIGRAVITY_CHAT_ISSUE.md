# Звіт про проблему: Несправність Agent чату в Antigravity IDE

## 🚩 Опис проблеми
При спробі відправити повідомлення в **Agent чат** (Gemini/Claude) всередині Antigravity IDE, запит миттєво завершувався без текстової відповіді, супроводжуючись лише звуковим сповіщенням про виконання завдання. Проблема була локалізована лише для конкретного проекту (`ways2spain-website`).

## 🔍 Технічна причина
Проблема виникла через несумісність внутрішнього мовного сервера Antigravity IDE з розширеними налаштуваннями Git-репозиторію.

1. **Версія репозиторію:** Використання інструментів (наприклад, **Claude Code**), що створюють **Git Worktrees**, автоматично оновлює `core.repositoryformatversion` з `0` до `1`.
2. **Розширення worktreeconfig:** Вмикається параметр `extensions.worktreeconfig = true`.
3. **Наслідок:** Мовний сервер Antigravity (`language_server`) не підтримує ці розширення Git. При спробі просканувати проект він видає помилку `core.repositoryformatversion does not support extension: worktreeconfig`. Через це "контекст проекту" (Workspace Info) стає порожнім (`nil`), і AI-агент не може згенерувати відповідь.

## 🛠 Як це трапилося
Проблема була спровокована використанням **Claude Code Desktop**, який при створенні сесій з worktrees автоматично змінює конфігурацію Git-репозиторію на рівні проекту.

## ✅ Вирішення (Fix)
Для відновлення роботи чату необхідно повернути Git-репозиторій до стандартного формату, який розуміє IDE:

```bash
# 1. Видалити розширення worktreeconfig
git config --unset extensions.worktreeconfig

# 2. Повернути версію формату репозиторію до 0
git config core.repositoryformatversion 0
```

Після виконання цих команд та перезапуску IDE, мовний сервер знову може зчитувати структуру проекту, і чат починає працювати належним чином.

## 💡 Поради на майбутнє
- Якщо ви плануєте часто використовувати **Claude Code** паралельно з **Antigravity**, пам'ятайте, що створення worktrees знову змінить цей конфіг.
- У разі повторного "мовчання" чату Gemini — першим ділом перевіряйте лог за шляхом:
  `~/Library/Application Support/Antigravity/logs/<сесія>/window1/exthost/google.antigravity/Antigravity.log`
  на наявність помилок `worktreeconfig`.
